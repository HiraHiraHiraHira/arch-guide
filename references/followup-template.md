# 独立专题 followup 模板

建议把最终成稿写入本地 Markdown 文件。若用户没有指定文件名，默认使用 `<project-name>-<topic>-followup.md`。如果当前工作区已经有同一研究对象的主报告或 followup，优先沿用其所在目录，不要为了专题落盘额外创建目录体系。

## 写作约定

- 这是一份单一专题文档，只处理一个主题，不重写主报告。
- 开头要明确写出：本专题在回答什么，它与主报告是什么关系，主报告文件是什么。
- 默认不要复用 `这份文档怎么读`、`5 分钟认识这个系统`、`项目在解决什么问题` 这类全局入口章节。
- 专题图优先用 Mermaid；如果流程图不适合 Mermaid，就改用编号流程，不要改成 ASCII 流程图。
- 如果这轮产出了会改变整体理解的结论，结尾要明确列出“建议回灌到主报告”的条目。
- 代码块后面紧跟一句到几句解释，回答“这段代码在当前专题里负责什么”。
- 行文直接写专题事实、动作和判断；避免 `不是……而是……`、`换言之`、`这意味着`、`为了方便理解`、`本文将` 这类套话。
