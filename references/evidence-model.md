# 证据模型

## 三层证据

### 1. 直接证据
- 来自代码、配置、数据、日志、接口定义、原始文本或明确可观察行为
- 可以被重新定位和复核

### 2. 工程推断
- 基于直接证据和系统设计经验得出的解释
- 必须说明推断依据，不可伪装成事实

### 3. 外部支持
- 来自官方文档、论文、成熟项目或权威技术材料
- 用于说明某类设计有何公共支持，不等于证明本项目一定成功

## 常见混淆

- 把 README 口径写成实现事实
- 把“作者大概率想做什么”写成“作者明确写了什么”
- 把外部项目的成功写成当前项目已验证

## 使用规则

1. 先记直接证据，再写推断。
2. 外部支持要说明“支持了什么”与“没有证明什么”。
3. 当证据不足时，明确写出不确定性。
