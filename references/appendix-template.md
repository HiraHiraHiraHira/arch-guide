# 材料补充 appendix 模板

建议把最终成稿写入本地 Markdown 文件。若用户没有指定文件名，默认使用 `<project-name>-<topic>-appendix.md`。如果当前工作区已经有同一研究对象的主报告、followup 或 appendix，优先沿用其所在目录，不要为了材料整理额外创建目录体系。

## 写作约定

- 这是一份材料型补充文档，用来承载清单、矩阵、索引、对照表或专项导读。
- 默认不要把它写成长篇叙事报告，不要重讲整套系统。
- 开头要说明：这份附录在整理什么、适用范围是什么、与主报告或 followup 的关系是什么。
- 证据、配置、接口、模块映射等内容优先用表格或列表组织。
- 如果有使用限制、采样边界或不完整性，要显式写出。
- 行文直接写材料内容和适用边界；避免 `不是……而是……`、`换言之`、`这意味着`、`为了方便理解`、`本文将` 这类套话。
