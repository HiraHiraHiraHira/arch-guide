# 架构导览与研究报告模板

建议把最终成稿写入本地 Markdown 文件。若用户没有指定文件名，默认使用 `<project-name>-report.md`；项目名无法可靠判断时，使用 `architecture-report.md`。如果当前工作区已经有同主题主报告，优先沿用原目录，不要为了落盘额外创建目录体系。

## 写作约定

- 项目内术语第一次出现时，写成 `AgentLoop（主循环）` 这种格式。
- 图示标签优先给读者语言注释，不要只堆英文名。
- Mermaid 图优先使用兼容性更高的简单流图。
- 总览结构图如果不适合 Mermaid，就改用分层列表或部件表；只有极小静态结构才使用 ASCII。
- 流程图如果不适合 Mermaid，就改用编号流程，不要改成 ASCII 流程图。
- 代码块后面紧跟一句到几句解释，回答“这段代码在系统里负责什么”。
- 不要在正文里单独声明语言策略、写作原则或面向对象；让这些约束直接体现在文风里。
- 行文直接写事实、动作和判断；避免 `不是……而是……`、`换言之`、`这意味着`、`为了方便理解`、`本文将` 这类套话。
